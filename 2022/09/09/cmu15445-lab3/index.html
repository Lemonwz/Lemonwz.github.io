cmu15445 project#3 - Query Execution

Project #3: Query Execution

这部分实验要求为DBMS添加对查询执行的支持，根据给定的查询计划执行对应的查询操作

实验地址：https://15445.courses.cs.cmu.edu/fall2021/project3/

Task #1: Executors

实验使用的处理模型是Iterator Model（也称Pipeline Model），查询计划中的每个操作符都提供了一个Next函数接口供上层操作符调用，Next函数返回单个元组或返回表示没有更多可用元组的标识符。操作符节点调用其子操作符的Next函数获取输入元组，处理后生成输出元组作为自身Next函数的返回值向上传递；叶子节点操作符从磁盘读入元组向上传递

实验要求实现9个操作符：顺序扫描、插入、更新、删除、嵌套循环连接、哈希连接、聚合、Limit和Distinct，对于每一个操作符，都有一个对应的PlanNode类（可以视为操作符的元数据）和Executor类（实现了Init和Next函数，Init函数用于初始化操作符，Next函数在每次调用时返回一个元组和对应的RID）

实验中涉及到的一些API类（也是Project3主要难点：对这些helper classes的理解）：

  • Value：表中数据的抽象类，所有Value对象都包含类型和比较函数，不同子类有着不同的实现；常用函数：
    • GetAs()：将Value对象强制转换为指定类型对象返回
    • CompareEquals(const Value &o)：比较两个Value对象是否相同，返回true/false
    • Add()、Subtract()、Min()、Max()：Value对象相加/相减，返回两个Value对象中较小/较大的一个
  • Tuple：表中的一行数据，data_字段存放的是实际数据，rid_字段仅在指向TableHeap时有效（即通过TableIterator获取时），常用函数：
    • Tuple(std::vector<Value> values, const Schema *schema)：通过Value数组和Schema构造Tuple
    • GetValue(const Schema *schema, uint32_t column_idx)：返回指定列的Value
    • KeyFromTuple()：根据参数返回一个Key Tuple用于索引相关操作
  • Column：表中属性（列）的元数据，包括属性名、类型、表达式等，常用函数：
    • GetExpr()：返回属性对应的AbstractExpression对象
  • RID：Record ID，用于记录Tuple的物理地址（物理页ID以及页中槽位）
  • Schema：对表/元组的格式说明（包含哪些属性），主要包含一个Column对象数组，主要函数：
    • GetColumns()：返回Column对象数组
    • GetColumnCount()：返回Schema中包含的Column数量
  • Index：索引，通过project2实现的ExtendableHashTable将Tuple映射到RID，主要函数：
    • GetKeySchema()：返回索引包含的Column组成的Schema对象
    • GetKeyAttrs()：返回索引包含的Column的col_idx数组
    • InsertEntry()：向索引中插入一个Tuple-RID键值对
    • DeleteEntry()：从索引中删除一个Tuple-RID键值对
  • Catalog：表格和索引的全部信息，提供对表格和索引的查询创建函数，主要函数：
    • GetTable()：通过table_name或table_oid获取表格信息（TableInfo对象）
    • GetTableIndexes()：通过table_name获取表格所有索引
  • TableHeap：表的物理表示，实际是一个存放TablePage的双向链表（由project1实现的BufferPoolManager管理），提供对元组的修改操作，主要函数：
    • InsertTuple()：插入元组，如果元组过大（超过PAGE_SIZE）则返回false
    • MarkDelete()：标记元组为删除状态（调用ApplyDelete函数时统一删除）
    • UpdateTuple()：更新元组，如果新元组过大（原先页面放不下）则返回false
  • TableIterator：TableHeap的迭代器，重载了部分运算符，可直接用做Tuple的指针：
    • const Tuple &operator*()：返回迭代器指向的Tuple对象
    • TableIterator &*operator*++()：返回自增后的迭代器
    • TableIterator operator++(int)：返回自增前的迭代器
  • AbstractExpression：表达式抽象类，包含三个纯虚成员函数：Evaluate()、EvaluateJoin()、EvaluateAggregate()，四个派生类对应四种类型的表达式：
    • ConstantValueExpression：常量表达式，三个成员函数均返回这个常量
    • ColumnValueExpression：属性值表达式，Evaluate函数返回元组指定列的Value，EvaluateJoin函数根据tuple_idx_值返回左元组或右元组指定列的Value（tuple_idx_为0返回左元组指定列Value，为1返回右元组指定列Value）
    • AggregateValueExpression：聚合表达式，只提供EvaluateAggregate函数，根据is_group_by_term_值返回group by对应列Value或aggregate对应列Value
    • ComparisonExpression：比较表达式，返回两个子表达式值的比较结果，子表达式的可能组合如下：
      1. 常量表达式+属性值表达式：Evaluate函数将元组的列值与常量比较（WHERE语义）
      2. 常量表达式+聚合表达式：EvaluateAggregate函数将元组GROUP BY属性值或聚合结果与常量比较（HAVING语义）
      3. 属性值表达式+属性值表达式：EvaluateJoin函数比较两个元组的两个属性值是否相同（JOIN ON语义）
  • AbstractPlanNode：查询计划由PlanNode组成，每个PlanNode对应一个操作符，PlanNode可以视为操作符的元数据，PlanNode抽象类包含操作符的输出格式output_schema_和子操作符数组children_，不同操作符的PlanNode类实现不同
  • AbstractExecutor：操作符抽象类，每个操作符都包含一个ExecutorContext对象，表示操作符执行的上下文信息，并需要实现Init和Next函数
  • ExecutionEngine：操作符执行引擎，先调用操作符的Init函数，随后循环调用Next函数，Next函数返回true时将输出元组加入到result_set，返回false时停止调用并返回result_set，循环调用操作符Next函数时，ExecutionEngine还会捕获调用过程中抛出的异常，如果检测到异常则停止调用

1. SEQUENTIAL SCAN：

SeqScanExecutor遍历指定表，每次返回一个元组（如果扫描得到的元组不满足给定谓词，则跳过该元组）

  • Init函数通过table_oid获取表table_，构造迭代器iter_指向table_表头
  • 如果迭代器iter_指向表尾，表示遍历结束，返回false
  • 遍历output_schema_中的Column数组，调用Evaluate函数获取iter_指向元组每个Column的Value，存入values数组
  • 使用values数组和output_schema_构建元组result_tuple
  • 如果result_tuple满足谓词predicate_（其实就是一个ComparisonExpression），设置tuple为result_tuple并返回true，否则跳过这个元组

2. INSERT：

InsertExecutor将元组插入到指定表中并更新表索引，插入元组可能由InsertPlanNode直接给出（raw_values_字段），也可能来自子操作符，通过IsRawInsert函数判断

  • Init函数通过table_oid获取table_info_，如果IsRawInsert函数返回false，则调用子操作符的Init函数
  • 如果是Raw Insert，则直接从plan_读入待插入元组，调用InsertAndUpdate函数插入元组并更新索引
  • 否则循环调用子结点Next函数获取待插入元组，并对每一个元组执行如下操作（直到子操作符Next函数返回false，表示没有更多待插入元组）
    1. 调用InsertTuple函数将元组插入到表中，InsertTuple函数返回false表示元组大小超过了PAGE_SIZE，抛出异常并返回，返回true表示插入成功
    2. 更新表的所有索引（新增索引项索引当前新插入的元组）
  • Next函数应始终返回false，否则ExecutionEngine会将返回元组插入到result_set中，而插入操作不涉及对返回元组的赋值

3. UPDATE：

UpdateExecutor更新指定表中的现有元组以及表索引，待更新元组始终来自子操作符，update_attrs_用于将col_idx映射到UpdateInfo，UpdateInfo包括更新类型（Add或Set）和更新值update_val_

  • Init函数初始化子操作符，通过table_oid获取table_info_
  • 循环调用子操作符的Next函数，对每一个待更新元组old_tuple执行如下操作（直到子操作符Next函数返回false，表示没有更多待更新元组）
    1. 调用GenerateUpdatedTuple函数获取更新后的元组updated_tuple
    2. 调用UpdateTuple函数更新表中对应元组，UpdateTuple函数返回false表示更新后的元组无法放入old_tuple的页面中，抛出异常
    3. 删除表中所有old_tuple相关的索引项
    4. 新增指向updated_tuple的索引项
  • Next函数应始终返回false，否则ExecutionEngine会将返回元组插入到result_set中，而更新操作不涉及对返回元组的赋值

4 DELETE：

删除操作和插入更新操作大同小异，从子操作符接收待删除元组，调用MarkDelete函数将其标记为已删除状态即可

5. NESTED LOOP JOIN：

NestedLoopJoinExecutor通过嵌套循环连接将来自两个子操作符的元组组合在一起，谓词predicate_用于判断两个元组的连接属性是否相同（是一个比较表达式，EvaluateJoin函数返回比较结果），result_tuples_用于存放组合后的元组（由于JOIN操作可能会阻塞查询计划，因此这里选择在Init函数中完成待连接元组的获取和连接操作，将连接结果放入result_tuples_中，上层操作符调用Next函数时直接从result_tuples_获取即可）

  • 循环调用left_executor_的Next函数获取外部表元组left_tuple，对每个外部表元组执行如下操作：
    1. 初始化right_executor_，循环调用right_executor_的Next函数获取内部表元组right_tuple
    2. 对每个内部表元组，调用谓词predicate_的EvaluateJoin函数判断left_tuple和right_tuple能否JOIN
    3. 如果能则对output_schema_的每一个Column调用其EvaluateJoin函数获取属性值并添加到values数组
    4. 通过values数组和output_schema_构造元组并放入到result_tuples_中
  • 调用Next函数时，如果result_tuples_为空，说明没有剩余输出元组，返回false；否则从result_tuples_中取出一个元组，返回true

6. AGGREGATION：

AggregationExecutor将来自单个子操作符的多个元组聚集组合成一个元组，支持COUNT、SUM、MIN、MAX四种类型的聚集操作，同时还支持GROUP BY和HAVING子句；having_字段是比较表达式，用于对输出元组进行筛选；group_bys_是属性值表达式，用于对输入元组进行分类；aggregates_是属性值表达式，用于获取执行聚集操作的属性值；agg_types_表示聚集操作的类型

  • 如下所示，AVG(gpa)表示对输入元组的gpa属性取平均值，分别对应agg_types_和aggregates_字段
  • GROUP BY sid表示对输入元组按sid属性划分，对应group_bys_字段
  • HAVING avg_gpa > 3.9表示筛选出平均绩点大于3.9的输出元组

AggregationExecutor实现了自定义的哈希表SimpleAggregationHashTable，底层是unordered_map数据结构，AggregateKey结构体存放group_bys_字段对应的属性值，AggregateValue结构体存放aggregates_字段对应的属性值

  • 初始化子操作符，循环调用子操作符的Next函数获取child_tuple，获取child_tuple对应的AggregateKey和AggregateValue，调用InsertCombine插入到哈希表中
  • 初始化哈希表迭代器指向哈希表表头
  • Next函数获取当前迭代器指向元素的key和value，如果没有HAVING约束或当前键值对满足HAVING约束则生成聚集后的元组并返回true

7. HASH JOIN：

实现一个类似于AggregationExecutor中的哈希表以支持哈希连接操作，与嵌套循环连接输入输出相同，但连接策略不同（具体见lecture11）

  • Init函数初始化子操作符，循环调用left_executor_的Next函数获取left_tuple插入到哈希表中
  • 随后循环调用right_executor_的Next函数获取right_tuple，对每一个right_tuple获取其哈希值找到对应的left_tuple集合
  • 对left_tuple集合中的每一个left_tuple检查是否满足连接条件，如果满足则对output_schema_的每一个Column调用其EvaluateJoin函数获取属性值并添加到values数组，构造输出元组添加到result_tuples_，不满足则检查下一个left_tuple
  • 调用Next函数时，如果result_tuples_为空，说明没有剩余输出元组，返回false；否则从result_tuples_中取出一个元组，返回true

8. LIMIT：

LimitExecutor限制来自子操作符输出元组的最大数量，由limit_字段表示，cur_recv_字段表示当前已接收元组数量

当前已接收元组数量大于limit_时返回false

9. DISTINCT：

同样需要实现一个类似于AggregationExecutor中的哈希表，提供DistinctKey到Tuple的映射，其中DistinctKey是元组所有属性值组成的Value数组，Tuple是元组本身，试图插入哈希表时，会检查表中是否存在DistinctKey相同的元素，如果存在则说明这是一个重复的元组，跳过即可，否则插入到哈希表中，最后哈希表中剩余的元组就是去重之后的元组

  • Init函数初始化子操作符，循环调用子操作符的Next函数读入child_tuple
  • 获取child_tuple的DistinctKey，插入DistinctKey-child_tuple键值对到哈希表中，初始化迭代器指向表头
  • 调用Next函数时，如果迭代器指向表尾，表示所有去重后的元组均已输出，返回false，否则取出元组，迭代器递增
